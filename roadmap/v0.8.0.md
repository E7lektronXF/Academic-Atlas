# Roadmap — v0.8.0

**Status:** Planned
**Theme:** Discoverability + Freshness
**Starting point:** v0.7.0 (53 records, 8 categories)

This document is the agreed plan for the v0.8.0 release. It follows the
maintenance philosophy in `AI_CONTEXT.md`: incremental, documented, and
non-destructive. No existing verified record is removed or renumbered.

---

## Goals

Focus on the two things a student actually acts on — **when to apply** and
**what matches my interests** — plus a personal shortlist so they can build
their own selection.

Scope agreed for this release:

* **A — Subject tagging** (new schema field + site filter)
* **B — Realistic dates** (`Typical_Window` field + a verification sweep)
* **C — Shortlist / favorites** (client-side, `localStorage`)
* **Data growth** — balanced, ~+15 verified records (≈68 total)

**Out of scope (deferred to v0.9.0):** the scheduled "maintenance reminder"
GitHub Action that flags stale/expired records. Noted below under *Deferred*.

---

## 1) Schema changes (2 new columns, appended)

The column-order rule means both new columns are appended at the end
(current 17 → 18, 19).

| #  | Column           | Type                        | Required | Description |
|----|------------------|-----------------------------|----------|-------------|
| 18 | `Subject`        | string (multi, `;`-separated) | **Yes**  | Discipline tag, independent of `Category`. One or more values from the controlled vocabulary below. |
| 19 | `Typical_Window` | string                      | No       | When the application window usually opens, for records whose exact next date is not yet published. e.g. `September–October`, `Spring`. |

### Proposed `Subject` vocabulary

Rendered as chips on the site; documented as a table in `docs/DatabaseSchema.md`:

`Mathematics` · `Physics` · `Chemistry` · `Biology` · `Computer Science` ·
`Engineering & Robotics` · `Earth & Space` · `Environment` ·
`Economics & Business` · `Social Sciences & Humanities` · `Linguistics` ·
`Writing` · `Arts & Design` · `Medicine & Health` · `Interdisciplinary`

Notes:

* `Interdisciplinary` acts as a safe fallback so every record can carry at
  least one subject and the "required" rule never forces a bad fit.
* Multi-value separator is `;` (avoids ambiguity with commas inside values).

---

## 2) Validator & pipeline (`scripts/validate.py`)

* Add `Subject`, `Typical_Window` to `EXPECTED_HEADERS`.
* Add a `VALID_SUBJECTS` set + multi-value parsing: split on `;`, strip each
  part, check each is in the vocabulary and not duplicated within a record.
* Add `Typical_Window` to `OPTIONAL_COLUMNS` (`Subject` stays required).
* `Typical_Window` has no hard rule; only a length warning (> 40 chars).
* `scripts/xlsx_to_json.py` needs **no change** — the pipeline carries new
  columns into `docs/data.json` automatically.

---

## 3) Website (`docs/` — `app.js`, `style.css`, `index.html`)

**Subject filter**

* Clickable subject chips on cards and in the detail view; a subject selector
  in the browse toolbar.
* New `?subject=` URL parameter, consistent with the existing shareable-state
  pattern: update `matchesFilters`, `readUrl`/`writeUrl`, `syncControls`.

**Realistic date indicator**

* `deadlinePill`: when `Application_Deadline = UNKNOWN` but `Typical_Window`
  is set, show a specific pill (e.g. `🗓 Usually September–October`) instead
  of the generic "see notes". Add a matching row in the detail view.

**Shortlist / favorites** (no backend)

* ⭐ toggle → `localStorage` (`aa:shortlist`, array of IDs), with
  `aria-pressed` on the toggle.
* A **"My list (N)"** entry point in the header → a view showing only starred
  records (`?view=list`), with an empty state.
* **Sharing:** `?ids=COMP-0001;RES-0002…` renders a read-only shared list plus
  an "add all to my list" action. Filter out IDs that no longer exist.

## 4) Data work (~+15 records + verification sweep)

**Backfill:** assign `Subject` to all 53 existing records.

**Verification sweep (heaviest task):** re-check each record against its
official source → if the 2026–27 cycle date is published, set a concrete
`Application_Deadline`; otherwise fill `Typical_Window`; update
`Last_Verified`. Goal: sharply reduce the "bare see notes" cases.

**Growth (candidates — each to be verified against official sources):**

| Category               | Now | Target | Candidate pool (verify before adding) |
|------------------------|-----|--------|----------------------------------------|
| Academic courses       | 3   | ~6     | AoPS, Stanford Pre-Collegiate, edX/MITx high-school-relevant courses |
| Conferences & events   | 3   | ~6     | Additional MUNs, youth science summits |
| Research programs      | 4   | ~7     | Polygence, Lumiere, Horizon (paid → mark `Cost`) |
| Journals & pubs        | 4   | ~7     | STEM Fellowship, Curieux, Young Scientists Journal |
| 🇹🇷 National pathways   | 8   | ~11    | TEKNOFEST competitions, Deneyap, additional TÜBİTAK stages |

Candidates are not final — per project rules each must be confirmed against an
official source, and anything that doesn't meet the quality bar is dropped.

## Suggested sequence (phases)

1. **Phase 1 — Schema (code, no data):** schema doc + two empty headers in the
   xlsx + `validate.py`. *(At this point `Subject` is empty, so the validator
   intentionally reports "required empty"; it turns green once Phase 2 fills
   it.)*
2. **Phase 2 — Data:** backfill `Subject` for 53 records → verification sweep →
   ~+15 new records → run `xlsx_to_json` + pass `validate`.
3. **Phase 3 — Website:** subject filter, `Typical_Window` pill, favorites +
   sharing.
4. **Phase 4 — Docs & release:** update docs + `CHANGELOG` + (optionally) a
   PR, with CI green.

---

## Definition of done

* `validate.py` passes with 0 errors (~68 records); no CI drift.
* Site: subject filter works and is shareable (`?subject=`), `Typical_Window`
  pill shows, favorites persist and are shareable via `?ids=`.
* Column order preserved; no existing record altered or removed.

---

## Open decisions

* **Is `Subject` required?** Recommended: **required** with an
  `Interdisciplinary` fallback (keeps data clean). Alternative: start optional.
* **`Typical_Window` language:** recommended **English** (`September–October`)
  to match the rest of the site.

---

## Deferred to v0.9.0

* Scheduled GitHub Action that opens a "maintenance" issue listing records with
  `Last_Verified` older than 180 days or a passed deadline — fits the
  "AI-maintainable" identity.
* SEO: per-record static pages / JSON-LD structured data / `sitemap.xml`.
* Manual light/dark theme toggle (currently OS-driven only).
